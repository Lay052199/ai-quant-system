# ai_quant_system 项目规则

1. 当前阶段只允许历史回测和模拟分析，不允许真实下单。
2. 所有代码必须模块化，禁止把全部逻辑写进 app.py。
3. 所有核心函数必须有中文注释和清晰 docstring。
4. 数据获取模块放在 data/。
5. 策略信号模块放在 strategy/。
6. 回测引擎放在 backtest/。
7. 风险控制模块放在 risk/。
8. 报告生成模块放在 report/。
9. 所有交易计算必须考虑手续费。
10. 禁止未来函数：当天产生的信号，默认下一交易日执行。
11. 禁止把 API Key、Token、账号密码写进代码。
12. 后续如接入 API Key，必须使用 .env 或环境变量。
13. 每次新增重要功能后必须更新 README.md。
14. 图表标题、坐标轴、图例尽量使用中文。
15. 代码要适合 Windows + VS Code + PowerShell 环境运行。
16. 任何实盘相关功能只能作为预留接口，不允许实际执行交易。
